## **Parametric models** 

- Generally speaking, shape descriptors are simplified representations of 3D objects to describe geometric or topological characteristics of the 3D shape.

   - › However, the output is limited by the space that can be approximated with the chosen family of functions

## **PointNet principles** 

- **Permutation (order) invariance** 

   - › A point cloud of N points has N! permutations, which all represent the same shape 

   - › The network output must be the same (invariant) to the particular order 

- **Geometric transformation invariance** 

   - › The learned representation of the point set should be invariant to rigid transformations (rotation, translation) 

- **Interactions among points** 

   - › Neighboring points form a meaningful subset 

   - › The representation needs to capture local structures from nearby points, and the interactions among them 

   - › Important for pixel-level predictions (segmentation)

# Representing animations 

## **Animation** 

An animation or motion clip 𝑚∈ℝ[𝑑×𝑡] is defined by a temporal sequence of _t_ poses Poses can be expressed as a combination of the following 

- › as features of a parametric model 

- › as actions variables (e.g., Idle, Move, Control) which are then converted to an animation and blended 

- › as the positions of _J_ joints in space 

- › (more common) as the relative positions (rotations and velocities) of the _J_ joints at time _t_ w.r.t to the root position 

- › as a graph with _J_ nodes (joints) 

- › as interaction vectors/position of contact points
