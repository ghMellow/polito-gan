## **Why CLIP is fundamental for visual genAI?** 

- CLIP's text encoder is the conditioning backbone inside Stable Diffusion — it converts prompts into the vectors that guide image generation 

- Compare an image embedding against text embeddings of class labels 

   - (e.g. "a photo of a cat") — no fine-tuning needed 

      - Powers zero-shot classification without any task-specific training data 

      - Enables image-text retrieval at scale 

- CLIP score (cosine similarity between generated image and text prompt) is a standard automatic evaluation metric for text-to-image models 

- Nearly every topic in this course — diffusion models, latent diffusion, RL from human feedback — touches CLIP in some way 

## **Variants and limitations** 

- Limitation of CLIP: 77-token hard limit on caption. 

   - Comes from the fixed positional embedding table in the original Transformer text encoder (inherited from GPT-2) 

   - Long or complex prompts in Stable Diffusion are silently truncated at token  words beyond that threshold have zero influence on generation 

- Several variants extend the original CLIP: OpenCLIP (open-source reproduction), ALIGN (larger noisier data), SigLIP (sigmoid loss instead of softmax, more stable at large batch sizes), FLAVA (adds multimodal fusion head) 

- Workarounds: prompt weighting, prompt chunking (used in ComfyUI), or switching to architectures with longer context (e.g. SD 3 uses T5-XXL with up to 256 tokens)
