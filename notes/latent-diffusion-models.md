# Latent Diffusion Models (L-Diffusion) - Continuation

This part details how the model is guided by external information (like text prompts) and how the final image output is produced.

## Conditional Generation (Text-to-Image)
The ability to guide generation via text (Conditioning) is crucial. L-Diffusion models achieve this by incorporating the text embedding into the denoising network $\epsilon_\theta$.

*   **Mechanism**: The text embedding (usually generated by a separate large language model like CLIP) is integrated into the cross-attention layers of the U-Net architecture.
*   **Effect**: At every step of denoising, the model's prediction is conditioned not just on the noisy image state $z_t$, but also on the semantic meaning encoded by the prompt.

## The Sampling (Reverse) Process
The actual image generation is the reversal of the diffusion process, executed iteratively:
1.  Start with pure noise $z_T$ in the latent space.
2.  For $t = T$ down to 1:
    *   The model predicts the noise $\epsilon_\theta(z_t, t, c | \text{prompt})$.
    *   This predicted noise is used to estimate the less noisy state $z_{t-1}$, incorporating $\text{Prompt}$ conditioning ($c$).
3.  The final result $z_0$ is a clean, synthesized latent image.

## Decoding to Pixels
The synthesized latent code $z_0$ is passed one final time through the decoder part of the initial **VAE**. This reconstructs the $z_0$ latent representation back into the high-resolution pixel space, yielding the final image $x_{generated}$.

*The entire process is a pipeline: $\text{Text} \rightarrow \text{Conditioning} \rightarrow \text{Diffusion in Latent Space} \rightarrow \text{VAE Decoder} \rightarrow \text{Image} $.*
